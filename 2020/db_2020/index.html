数据库2020期末复习 (In Progress)

数据库系统概述

数据库，数据库管理系统（DBMS），数据库技术，数据库系统

基本概念

数据：具有一定格式（或结构）的符号串。

数据的特性

  • 数据表现的多样性
  • 数据的可构造性（型 type + 值 value）
  • 数据的挥发性/持久性（transient / persistent）
  • 数据的私有性/共享性（private $\rightarrow$ share）
  • 数据的“量” $\Rightarrow$ 少量/大量/海量/大数据

数据库用户：最终用户（终端查询用户、应用程序的使用者），应用程序开发人员，数据库管理员

数据库系统的发展及趋势

  1. 层状/网状数据库
    • 统一的数据管理工具，数据共享能力
    • 受文件的物理影响大；数据模式构造烦琐。
  2. 关系数据库（OLTP）
    • 结构简单、使用方便、逻辑性强、物理少
    • 模型的描述能力不足；数据操纵功能有待进一步扩展。（专业性有余而通用性不足）
  3. 数据仓库（OLAP）
  4. 新型数据管理系统

数据库系统的基本特点

  1. 集成性：统一的数据结构；全局统一的数据模式；根据每个应用的数据需要构作局部模式。
  2. 高共享性与低冗余性：减少不必要的存储空间；避免数据的不一致性。
  3. 独立性：数据库中的数据与使用这些数据的应用程 序之间的互不依赖性，即数据或数据结构的 改变不会导致对使用这些数据的应用程序的 修改，反之亦然。（物理独立性 + 逻辑独立性）
  4. 统一管理与控制：数据的完整性检查；数据的安全性保护；并发控制；数据库故障恢复。

数据库内部结构体系

  1. 概念模式（模式）
  2. 外模式（子模式、用户模式）
  3. 内模式（物理模式）
  4. 两级映射
    • 概念模式到内模式：实现物理独立性
    • 外模式到概念模式：实现逻辑独立性
